# 出行協助功能介紹

本文說明瞳伴（Tonbo）應用中「出行協助」模塊的用途、流程、技術接入與使用方式。

---

## 一、功能概述

出行協助為視障用戶提供**語音輸入目的地 → 步行路線規劃 → 分段語音導航 + 可步行路徑分析**的一站式出行輔助。

- **路線規劃**：基於**高德地圖** Web 服務 API，獲取步行路線步驟與總距離/時長。
- **導航播報**：TTS 分段播報「前行 XX 米」「左轉」「右轉」；接近轉彎時（約 20 米內）自動播報下一步指令。
- **可步行路徑分析**：導航頁相機 + 本機 **YOLO** 檢測前方障礙，結合**可步行路徑分析**（左/中/右區域佔比）給出「前方道路暢通」「中間受阻請向左/右慢行」「前方有斑馬線/紅綠燈」等語音提示，無需看地圖即可安全前行。

---

## 二、高德地圖接入狀態

### 2.1 已接入並可正常運行

| 類型 | 說明 | 狀態 |
|------|------|------|
| **高德 Web 服務 API** | 路線規劃與地理編碼（REST） | ✅ 已接入並在使用 |
| **高德 Android 地圖 SDK** | 導航頁地圖顯示、藍點定位、路線繪製 | ✅ 已接入 |

- **路線規劃**：`RoutePlanner` 通過 OkHttp 調用高德 Web 接口：
  - 地理編碼：`https://restapi.amap.com/v3/geocode/geo`（將目的地文字轉為經緯度）
  - 步行路線：`https://restapi.amap.com/v3/direction/walking`（起點→終點步行步驟與 polyline）
- **導航控制器**：`NavigationController` 已設置 `routePlanner.setApiProvider("amap")`，規劃結果來自高德。
- **地圖與定位**：`NavigationActivity` 內使用高德 MapView 顯示路線（藍線）、已走路段（灰線）、未走路段（藍線），並跟隨用戶定位；偏離路線時可觸發重新規劃。

### 2.2 注意事項

- **Web 服務 Key**：路線規劃使用 `RoutePlanner` 內配置的高德 **Web 服務** Key。若遇 `USERKEY_PLAT_NOMATCH` 等錯誤，需在高德開放平台申請「Web 服務」類型 Key 並替換。
- **網絡**：規劃與地理編碼需聯網；導航過程中的定位與 TTS、障礙檢測可在離線下進行（路線步驟已預先拉取）。

## 三、使用流程

```
主頁 → 出行協助 → 開始出行 → 語音/文字輸入目的地 → 確認 → 導航頁（路線播報 + 可步行路徑提示）
```

### 3.1 入口

- **主頁**：點擊「出行協助」卡片，進入 `TravelAssistantActivity`。
- **出行協助頁**：提供「開始出行」與「聯繫志願者」；點擊「開始出行」進入 `StartTravelActivity`。

### 3.2 開始出行（StartTravelActivity）

- **語音輸入**：點擊麥克風，說出目的地（如「我要去天水圍」），由 `VoiceCommandManager` 識別並解析為 `TravelParseResult`，填入編輯框。
- **文字輸入**：也可在編輯框中直接輸入或修改目的地。
- **確認出發**：點擊確認後進行位置權限檢查、跨區檢查（可選）及周邊 POI 解析；通過後跳轉導航頁，並傳入目的地字串與語言。

### 3.3 導航頁（NavigationActivity）

導航頁為**出行協助的主頁面**，整合路線導航與步行輔助。

| 功能 | 說明 |
|------|------|
| **路線規劃與播報** | 獲取當前位置後調用高德步行 API，得到分段步驟；TTS 播報「前行 XX 米」「左轉」等，支持「重報本段」「下一段」按鈕。 |
| **接近轉彎提醒** | 當用戶距離當前段終點約 20 米內時，自動播報該段指令（如「前方右轉」），並推進到下一段。 |
| **偏離路線檢測** | 實時位置與路線點距離超過閾值時提示「已偏離路線，正在重新規劃」，並以當前位置為起點重新規劃。 |
| **剩餘距離顯示** | 頁面頂部顯示「剩餘 XXX 米」「約 X 分鐘」，隨定位與當前步驟更新。 |
| **路線進度可視化** | 地圖上已走過的路段顯示為灰色，未走路段為藍色。 |
| **可步行路徑分析** | 相機 + YOLO 檢測行人、車輛、障礙物、斑馬線、紅綠燈等；分析屏幕下半部分左/中/右三區的「受阻程度」，並根據狀態播報：前方道路暢通、中間受阻請向左/右慢行、前方受阻請停下、前方有斑馬線、前方紅綠燈。 |
| **檢測框疊加** | 相機預覽上疊加 `OverlayView`，實時繪製檢測框與類別，便於理解前方場景。 |

路線播報與可步行路徑播報通過語音鎖協調，避免同時搶話。

## 四、可步行路徑分析（WalkAssistManager）

可步行路徑分析由 **WalkAssistManager** 負責，與 **YOLO 檢測**、**OverlayView** 一起構成導航頁的步行輔助層。

### 4.1 流程概覽

1. **TravelDetectionController** 綁定相機與 `ImageAnalysis`；當由 `NavigationActivity` 設置了步行輔助分析器時，每幀調用該分析器（不再使用控制器內建的舊障礙播報）。
2. **NavigationActivity** 在分析器內：使用 **YoloDetector** 對當前幀做檢測 → 將結果轉為屏幕坐標的 **Detection** 列表 → **OverlayView** 繪製檢測框 → 調用 **WalkAssistManager.analyzeWalkablePath()** 得到路徑狀態 → 調用 **WalkAssistManager.speak()** 在狀態變化且間隔足夠時播報。

### 4.2 路徑狀態與播報

- **輸入**：當前幀的檢測列表（含邊框與類別）、屏幕寬高。
- **規則**：僅考慮「步行相關障礙」（人、車、樹、桿、椅、斑馬線、紅綠燈等）；僅統計**屏幕下半部分**的障礙；按左 1/3、中 1/3、右 1/3 計算各區「受阻面積比」，並支持檢測框跨區的重疊比例分配。
- **輸出狀態**：`clear`（無障礙）、`center_clear`（中間可走）、`move_left` / `move_right`（中間受阻建議向左/右）、`blocked`（前方受阻）、`crosswalk`、`traffic_light`。
- **播報**：狀態變化且距上次播報超過 1.5 秒時，根據狀態播報對應短語（如「前方道路暢通」「中間受阻，请向左慢行」「前方有斑马线」等）。

### 4.3 相關類

| 類名 | 職責 |
|------|------|
| **WalkAssistManager** | 可步行路徑分析（左/中/右受阻計算）、狀態記憶、TTS 播報節奏與文案。 |
| **Detection** | 單一檢測結果（邊框坐標、類別、置信度），供 Overlay 繪製與路徑分析使用。 |
| **OverlayView** | 在相機預覽上繪製檢測框與類別標籤。 |
| **YoloDetector** | 本機 YOLO（TFLite）目標檢測，輸出行人、車輛、交通設施等。 |

## 五、主要類與職責

| 類名 | 職責 |
|------|------|
| **TravelAssistantActivity** | 出行協助入口頁；開始出行、聯繫志願者；多語言 UI。 |
| **StartTravelActivity** | 目的地輸入（語音/文字）、權限與跨區守衛、發起導航跳轉。 |
| **NavigationActivity** | 導航主頁；地圖、路線步驟、剩餘距離、轉彎提醒、偏離重規劃、路線進度可視化；集成 YOLO + WalkAssistManager + OverlayView 的可步行路徑分析與播報。 |
| **NavigationController** | 定位、調用 RoutePlanner（高德）、路線步驟/總距離/總時長回調、導航狀態與週期定位更新、偏離時觸發重規劃。 |
| **RoutePlanner** | 路線規劃；支持 Google / 高德；當前使用高德步行 API + 高德地理編碼；輸出步驟與 polyline。 |
| **TravelDetectionController** | 相機綁定與 ImageAnalysis；可選「步行輔助分析器」由 NavigationActivity 提供（YOLO + 路徑分析 + 播報），否則使用內建 YOLO 與舊版障礙播報。 |
| **WalkAssistManager** | 可步行路徑分析與狀態播報（見上）。 |
| **LocationService** | 跨區檢查、周邊 POI 查詢（用於目的地解析）。 |
| **VoiceCommandManager** | 語音識別與目的地短語解析（如「我要去 XXX」）。 |

## 六、多語言與無障礙

- 出行協助相關界面與路線 TTS 播報支持**粵語、英文、普通話**，與應用當前語言一致。
- 可步行路徑播報使用 **TextToSpeech** 與系統預設語言，文案為簡體中文短句。
- 按鈕點擊帶震動反饋；關鍵狀態（如「正在規劃路線」「路線已規劃」「導航中」）有語音提示；導航頁提供重報本段、下一段等操作，方便視障用戶獨立使用。

---

## 七、小結

- **高德地圖**：路線規劃與地理編碼已通過高德 Web 服務 API 正常連接；導航頁使用高德 MapView 顯示路線與定位，並支持偏離重規劃與剩餘距離顯示。
- **出行協助流程**：主頁 → 出行協助 → 開始出行 → 語音/文字目的地 → 確認 → 導航頁，實現「語音目的地 + 高德步行路線 + 分段 TTS 導航 + 轉彎提醒 + 可步行路徑分析」的完整流程。
- **步行輔助**：導航頁整合 YOLO 檢測與 **WalkAssistManager** 的可步行路徑分析，根據左/中/右受阻情況與斑馬線、紅綠燈等給出簡短語音提示，並在預覽上疊加檢測框，方便視障用戶在無需看地圖的前提下安全前行。
