# 瞳伴（Tonbo）项目总结与汇报材料

**项目名称**：瞳伴（Tonbo）  
**团队人数**：4 人  
**项目类型**：面向视障人士的综合智能辅助应用  

本文档适用于：PPT 制作、项目答辩、比赛展示、项目汇报。

---

## 一、项目总体介绍

### 1.1 项目背景：视障人群面临的生活问题

- **出行困难**：无法识别路牌、红绿灯、斑马线，易迷路或发生碰撞，独立出行风险高。
- **环境感知缺失**：难以识别周围物体（人、车、障碍物）、颜色与光线，影响安全与社交。
- **信息获取障碍**：无法阅读文档、票据、货币面额，日常办事与消费依赖他人。
- **紧急求助不便**：突发情况下一键求助、位置分享、联络家人/志愿者的能力不足。
- **操作门槛高**：多数 App 依赖视觉界面，视障用户难以独立完成复杂操作。

### 1.2 我们希望解决的核心痛点

| 痛点 | 我们的应对 |
|------|------------|
| 看不见周围有什么 | 实时 AI 物体检测 + 语音播报（人、车、障碍物、位置） |
| 不会认路、怕走错 | 语音目的地 → 步行路线规划 → 分段 TTS 导航 + 可步行路径分析 |
| 读不了文件、认不了钱 | OCR 文字识别 + 港币识别，语速/多语言朗读 |
| 紧急时难以求助 | 长按 3 秒紧急按钮、选联络人、自动发位置与短信、智能拨号 |
| 不想一直点屏幕 | 语音命令 + 连续对话 + LLM 智能对话，手势绑定快捷功能 |

### 1.3 App 整体定位

**瞳伴（Tonbo）** 是一款面向视障人士的 **综合智能辅助平台**，集「环境感知、阅读辅助、出行导航、紧急求助、语音交互、即时协助」于一体，通过 **计算机视觉 + 多语言语音 + 地图与导航** 技术，在 **不依赖他人** 的前提下，提升视障用户的 **安全、独立与信息获取能力**。

### 1.4 项目社会价值与无障碍意义

- **平等参与**：让视障用户也能独立完成出行、阅读、求助等日常行为，减少对陪护的依赖。
- **安全与尊严**：实时环境与路面提示、紧急位置分享与一键求助，提升外出安全与心理安全感。
- **技术普惠**：将 AI、TTS、地图能力以「语音优先、触觉反馈、大按钮」的无障碍方式落地，具有可推广性。
- **社会效益**：可作为政府、公益组织、企业推进信息无障碍与智慧城市的参考案例。

## 二、App 整体功能模块介绍

以下按模块说明：**模块作用**、**技术实现方式**、**实际意义**。

---

### 2.1 首页与导航模块

- **作用**：统一入口、七大功能一键访问、多语言切换、紧急求助入口、震动手感反馈。
- **技术实现**：`MainActivity` + 卡片式布局、`LocaleManager` 三语言（粤/英/普）、`VibrationManager` 点击/长按反馈、长按 3 秒触发 `EmergencyManager`、固定顶部 + 可滑动内容。
- **实际意义**：降低认知负担，视障用户通过语音/触控即可快速进入任意功能；紧急场景下长按即可求助，减少误触。

---

### 2.2 环境感知模块

- **作用**：用摄像头「看见」周围物体与光线，通过语音告诉用户「有什么、在哪、亮还是暗」。
- **技术实现**：TensorFlow Lite **SSD MobileNet**（80 类 COCO）+ **多帧融合稳定性过滤**（连续 3 帧才播报）、置信度与边界框验证、智能播报优先级（人/车优先）、左/中/右位置描述、**夜间模式**（低光自动降阈值）、`ColorLightingAnalyzer` 颜色与光度分析、`OptimizedDetectionOverlayView` 绿/黄/红置信度框。
- **实际意义**：用户无需视力即可获知「前面有人/车」「左侧有椅子」「环境偏暗」等，提升行走与社交安全感；夜间模式改善低光下的可用性。

---

### 2.3 文件与货币阅读模块

- **作用**：识别文档文字与港币面额，并朗读给用户。
- **技术实现**：Google **ML Kit** 中英文 OCR、专用港币识别（纸币/硬币）、分按钮操作（文字 / 货币分开）、大范围扫描提示、TTS 朗读结果。
- **实际意义**：独立阅读信件、票据、药品说明；购物时确认钞票与找零，减少被欺骗或找错钱的风险。

---

### 2.4 紧急求助模块

- **作用**：一键触发求助：选联络人、发短信、发位置、拨打电话（含 999）。
- **技术实现**：长按 3 秒防误触、`EmergencyManager` 联络人列表、电话号**逐位数字朗读**（粤/英/普）、优先拨 999、位置权限与短信/拨号权限处理、自定义求助文案。
- **实际意义**：突发情况下一键通知家人或急救，并自动分享位置，缩短救援时间、降低沟通成本。

---

### 2.5 出行协助模块

- **作用**：从「说目的地」到「跟着语音走」的全流程：路线规划、分段导航播报、接近转弯提醒、偏离重规划，以及**可步行路径分析**（前方是否畅通、是否需左/右绕行、斑马线/红绿灯提示）。
- **技术实现**：  
  - **路线**：`RoutePlanner` + 高德 Web 服务（地理编码 + 步行路线）、OkHttp 请求；`NavigationController` 周期定位、偏离检测与重规划。  
  - **导航页**：`NavigationActivity` + 高德 MapView（蓝点、已走/未走路线）、TTS 分段播报「前行 XX 米」「左转」等，约 20 米内自动播报下一段。  
  - **语音入口**：`VoiceCommandManager` 解析「我要去 XXX」→ `TravelParseResult` → `StartTravelActivity` 填入目的地。  
  - **可步行路径**：`TravelDetectionController` + CameraX + **YOLO（TFLite）** 检测行人/车/障碍/斑马线/红绿灯；`WalkAssistManager` 分析屏幕下半部分左/中/右受阻占比，播报「前方畅通」「中间受阻请向左/右慢行」「前方有斑马线」等；`OverlayView` 绘制检测框。
- **实际意义**：视障用户可**仅凭语音**完成从输入目的地到步行导航的全过程，并结合前方障碍与交通设施提示，减少碰撞与迷路，实现「听得懂的导航」。

---

### 2.6 语音命令与智能对话模块

- **作用**：用自然语言控制 App（如「打开环境识别」「读文件」），以及连续对话、LLM 智能问答。
- **技术实现**：Android `SpeechRecognizer` 多语言（粤/英/普）、**模糊匹配**（Levenshtein、LCS、前缀等）提高容错、文本预处理与简繁转换、`VoiceCommandBuilder` 命令映射；**出行相关**：`parseDestinationFromTravelPhrase` 解析「我要去 X」并回调 `ExtendedVoiceCommandListener.onTravelDestinationRecognized`；**连续对话**：`StreamingVoiceAI` + `ConversationManager` 多轮上下文，**LLMClient**（DeepSeek/GLM-4-Flash）生成回复，不可用时回退关键词匹配；支持「查看聊天记录」「上一句/下一句」、长按切连续模式、语音唤醒/退出。
- **实际意义**：减少对屏幕触控的依赖，用「说人话」即可打开功能或问问题，出行场景下直接说目的地即可开始导航。

---

### 2.7 即时协助模块

- **作用**：一键联系志愿者、快速发预设消息、视讯通话。
- **技术实现**：`InstantAssistanceActivity`、Agora RTC 视讯、权限请求与说明、多语言界面与提示。
- **实际意义**：在需要真人协助时（如复杂路口、突发状况），可快速接通志愿者进行远程协助。

---

### 2.8 寻找物品模块

- **作用**：标记重要物品，之后用摄像头寻找并语音引导。
- **技术实现**：`FindItemsActivity`、物品标记与本地存储、相机实时比对、语音+震动反馈、位置记忆（可选）。
- **实际意义**：钥匙、遥控器等常用物品的查找，减少日常焦虑与时间消耗。

---

### 2.9 手势管理模块

- **作用**：为常用功能绑定自定义手势，画手势即可打开对应功能。
- **技术实现**：`GestureManagementActivity` + `GestureRecognitionManager`、手势绘制与模式匹配、与七大功能绑定、Intent 传递语言保持一致性。
- **实际意义**：为熟练用户提供快捷方式，减少层级点击，且不依赖视力。

---

### 2.10 系统设定模块

- **作用**：语速/音调/音量（0–200%）、震动手感、读屏与手势开关、多语言 TTS 选择、设定持久化与一键恢复默认。
- **技术实现**：`SettingsActivity`、`TTSManager` 参数应用、`SharedPreferences` 持久化、即时语音测试。
- **实际意义**：满足不同用户听力与操作习惯，提升长期使用舒适度。

---

### 2.11 用户与引导

- **作用**：登录/访客模式、启动画面与语音引导、每页进入时播报标题与操作说明。
- **技术实现**：`UserManager`、`SplashActivity`、`BaseAccessibleActivity.announcePageTitle()`、各页 `contentDescription` 与 TalkBack 兼容。
- **实际意义**：兼顾账户与隐私；首次使用和每页都有清晰语音引导，降低学习成本。

## 四、系统整体架构说明

### 4.1 应用分层

| 层级 | 职责 | 代表组件 |
|------|------|----------|
| **表现层** | 界面、语音播报、震动手感、无障碍 | MainActivity、各功能 Activity、BaseAccessibleActivity |
| **业务/控制层** | 流程编排、状态与权限 | NavigationController、EmergencyManager、VoiceCommandManager、TravelDetectionController |
| **能力层** | 视觉、语音、定位、网络、地图 | ObjectDetectorHelper、YoloDetector、TTSManager、RoutePlanner、WalkAssistManager、LLMClient |
| **数据/持久化** | 设定、用户、本地数据 | SharedPreferences、UserManager、本地标记与历史 |

### 4.2 核心数据流与调用关系

- **环境感知**：CameraX ImageAnalysis → ObjectDetectorHelper（SSD）/ YoloDetector → 多帧融合与优先级 → TTSManager 播报 + Overlay 绘制。
- **出行协助**：语音「我要去 X」→ VoiceCommandManager.parseDestinationFromTravelPhrase → StartTravelActivity → 高德地理编码 + 步行路线 → NavigationController 定位与偏离检测 → NavigationActivity 地图与 TTS；相机 + YOLO → WalkAssistManager 左/中/右分析 → 路况短语播报。
- **语音对话**：麦克风 → ASR → VoiceCommandManager / StreamingVoiceAI → 命令执行或 LLMClient → TTSManager 播报。

| **可选/扩展** | ONNX Runtime（可选 ONNX 模型）、Kotlin 协程 | 模型扩展、异步任务 |

## 六、创新点与亮点总结

1. **「听得懂的出行」全链路**：语音输入目的地 → 高德步行路线 → 分段 TTS 导航 + 接近转弯自动提醒 + 偏离重规划；结合 **可步行路径分析**（YOLO + 左/中/右受阻 + 斑马线/红绿灯），实现「不看地图也能安全走」的闭环。
2. **多帧融合 + 优先级播报**：环境检测连续 3 帧稳定才播报，人/车优先、左中右位置描述，减少误报与信息过载，更适合视障场景。
3. **夜间模式自动适配**：根据检测置信度推断低光，自动降低阈值并播报「夜間模式已開啟」，无需用户操作。
4. **语音与出行深度结合**：不仅支持「打开出行协助」，更支持「我要去天水围」等自然句直接进入路线与导航，降低操作步骤。
5. **连续对话 + LLM 回退**：支持多轮对话与 LLM 智能回复，网络不可用时自动回退关键词匹配，保证可用性。
6. **无障碍贯穿全产品**：大按钮、高对比、震动手感、电话号逐位朗读、三语言统一、TalkBack 兼容，从首页到出行、求助、设定均按视障习惯设计。
7. **多技术融合**：TFLite（环境 + 出行路面）+ ML Kit（OCR）+ 高德（路线与地图）+ TTS/ASR + LLM，在单 App 内形成「看、听、说、走」一体化的综合辅助平台。

## 七、完整用户使用流程

### 7.1 首次使用

1. 安装并打开瞳伴 → 启动页（SplashActivity）语音引导。  
2. 选择「登录」或「访客」→ 进入主页面（MainActivity）。  
3. 听语音介绍七大功能；可到「系统设定」调整语速/音量/语言（粤/英/普）。  
4. 在「紧急设置」中预先添加紧急联络人（可选但建议）。

### 7.2 日常典型流程

**环境感知**  
主頁 → 環境識別 → 允許相機權限 → 對準周圍 → 聽「左側有人」「中央有車」「環境偏暗」等；夜間會自動進入夜間模式並播報。

**阅读文档/认钱**  
主頁 → 閱讀助手 → 選擇「文字識別」或「貨幣識別」→ 對準文檔/紙幣 → 聽朗讀結果。

**出行（完整流程）**  
1. 主頁 → 出行協助 → 開始出行。  
2. 在「開始出行」頁：語音說「我要去 XXX」或文字輸入目的地 → 確認。  
3. 授予位置權限後，自動規劃步行路線並進入導航頁。  
4. 聽「前行 XX 米」「前方右轉」等；接近轉彎約 20 米內會自動重複下一段。  
5. 同時聽路面提示：「前方道路暢通」「中間受阻請向左慢行」「前方有斑馬線」等。  
6. 若偏離路線，會提示「已偏離路線，正在重新規劃」並重新規劃。  
7. 到達或中途可點「結束導航」退出。

**紧急求助**  
主頁 → 長按紅色緊急按鈕約 3 秒 → 選擇聯絡人（可多選）→ 自動發短信與位置、撥打 999 或第一聯絡人；電話號碼會逐位朗讀以便確認。

**语音控制**  
主頁 → 語音助手 → 說「打開環境識別」「讀文件」「我要去 XX」等；可長按進入連續對話模式，與 LLM 問答或發指令。

**手势快捷**  
主頁 → 手勢管理 → 為「出行協助」「環境識別」等綁定手勢；之後在支持頁面畫手勢即可直接進入對應功能。

### 7.3 流程小结（便于做 PPT 流程图）

- **环境感知**：主頁 → 環境識別 → 相機對準 → 語音播報物體與光線。  
- **阅读**：主頁 → 閱讀助手 → 文字/貨幣 → 對準 → 朗讀。  
- **出行**：主頁 → 出行協助 → 開始出行 → 語音/文字目的地 → 確認 → 導航頁（路線播報 + 可步行路徑提示）。  
- **紧急**：主頁 → 長按緊急按鈕 3 秒 → 選聯絡人 → 發送位置與短信、撥號。  
- **语音**：主頁 → 語音助手 → 單次指令或連續對話（含「我要去 X」）。
