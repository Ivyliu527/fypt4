### 第 3 張 — 專案目標
- **主要目標**：打造**一站式輔助應用**，幫助視障用戶更獨立地生活。  
- **範圍**：環境感知、文件／貨幣閱讀、**出行協助（導航＋可步行路徑分析）**、緊急求助、語音控制、即時志願者協助。  
- **成功標準**：語音優先、盡量少觸控；支持粵語、英語、普通話；無需視力即可使用（TTS、震動、大按鈕）。  

---

### 第 4 張 — 問題背景（1/2）
- **出行**：難以識別路牌、紅綠燈、斑馬線 → 易迷路或發生碰撞。  
- **環境**：難以知道周圍有什麼（人、車、障礙物、光線）。  
- **信息**：無法閱讀文檔、票據、紙幣 → 日常事務依賴他人。  

---

### 第 5 張 — 問題背景（2/2）
- **緊急**：難以一鍵求助、分享位置或快速聯絡家人／999。  
- **可用性**：多數 App 依賴視覺介面 → 視障用戶無法獨立操作。  
- **我們的回應**：用一個 App 同時解決**看（CV）、讀（OCR）、走（導航）、問（語音＋LLM）**。  

---

### 第 6 張 — 解決方案總覽：瞳伴提供什麼
- **環境識別**：實時物體檢測（80 類 COCO）＋ TTS（如「左側有人」「前方有車」）＋ 夜間模式。  
- **文件與貨幣**：OCR ＋ 港幣識別 ＋ 朗讀。  
- **出行協助**：語音目的地 → 步行路線（高德）→ 分段 TTS 導航 ＋ **可步行路徑分析**（障礙、斑馬線、紅綠燈）。  
- **緊急求助**：長按 3 秒 → 選擇聯絡人 → 發送短信 ＋ 位置 ＋ 撥號（如 999）。  
- **語音與 AI**：語音命令 ＋ 連續對話 ＋ LLM（DeepSeek／GLM-4-Flash），不可用時回退關鍵詞。

### 第 7 張 — 高階：使用案例（主要參與者）
- **參與者**：視障用戶。  
- **使用案例（示例）**：  
  - 識別環境（物體、位置、光線）。  
  - 閱讀文檔／識別貨幣。  
  - 規劃路線並以語音導航。  
  - 獲取可步行路徑提示（暢通／向左或右／斑馬線／紅綠燈）。  
  - 觸發緊急求助（短信 ＋ 位置 ＋ 撥號）。  
  - 用語音控制 App（如「打開環境識別」「去 X」）。  
  - 聯繫志願者（訊息／視訊）。  
- **可選**：簡單使用案例圖（用戶 — 上述用例）。

### 第 8 張 — 高階：系統架構（方塊圖）
- **表現層**：MainActivity、TravelAssistantActivity、NavigationActivity、設定等；BaseAccessibleActivity（TTS、震動、語言）。  
- **業務／控制層**：NavigationController、EmergencyManager、VoiceCommandManager、TravelDetectionController。  
- **能力層**：ObjectDetectorHelper、YoloDetector、TTSManager、RoutePlanner、WalkAssistManager、LLMClient。  
- **數據**：SharedPreferences、UserManager、本地存儲。  
- **建議**：一頁圖示，3～4 層，每層 2～3 個組件，無需代碼。  

---

### 第 9 張 — 關鍵設計：出行流程（簡化）
- 用戶說「我要去［地點］」或輸入目的地。  
- **RoutePlanner**（高德 Web API：地理編碼 ＋ 步行）→ 步驟 ＋ 折線。  
- **NavigationController**：GPS、步驟進度、偏離檢測 → 重新規劃。  
- **NavigationActivity**：地圖（高德 SDK）、TTS（「前行 50 米」「右轉」）、約 20 米轉彎提醒。  
- **WalkAssistManager**：相機 ＋ YOLO → 左／中／右佔比 → TTS（「前方暢通」「請向左」「前方斑馬線」）。  

---

### 第 10 張 — 關鍵設計：環境與語音
- **環境**：CameraX → SSD MobileNet／YOLO → 多幀穩定性（3 幀）→ 優先級（人／車優先）→ TTS ＋ 疊加層。夜間模式：根據置信度推斷低光時自動開啟。  
- **語音**：SpeechRecognizer（粵／英／普）→ 模糊匹配 ＋「我要去 X」解析 → 命令或出行目的地 → 執行或 LLM → TTS。  

---

### 第 11 張 — 類別圖（簡化，5～7 個類）
- **建議框**：  
  - `TravelAssistantActivity` → `StartTravelActivity`、`NavigationActivity`。  
  - `NavigationController` → `RoutePlanner`、`LocationService`。  
  - `NavigationActivity` → `WalkAssistManager`、`TravelDetectionController`。  
  - `VoiceCommandManager` → `TravelParseResult`、`ExtendedVoiceCommandListener`。  
- 一頁只畫類名與「使用／調用」箭頭，不需屬性。  

---

### 第 12 張 — 技術棧（簡要）
- **平台**：Android（minSdk 21）、Java 11。  
- **視覺**：CameraX、TensorFlow Lite（SSD ＋ YOLO）、Google ML Kit（OCR）。  
- **地圖**：高德 Web（地理編碼、步行）、高德 Android 3D SDK。  
- **語音**：Android TTS、SpeechRecognizer。  
- **網絡／AI**：OkHttp、Gson；DeepSeek／GLM-4-Flash（LLM）；Agora RTC（視訊）。

### 第 14 張 — 創新點與亮點（1/2）
- **「聽得懂的出行」全鏈路**：語音目的地 → 高德步行路線 → 分段 TTS ＋ 轉彎提醒 ＋ 偏離重規劃 ＋ **可步行路徑分析**（障礙、斑馬線、紅綠燈），用戶不看地圖也能安全走。  
- **多幀融合 ＋ 優先級**：環境需連續 3 幀才播報；人／車優先、左中右位置 → 減少誤報與信息過載。  

---

### 第 15 張 — 創新點與亮點（2/2）
- **夜間模式**：自動低光檢測（基於置信度）、自動調低閾值並播報「夜間模式已開啟」。  
- **語音與出行結合**：「我要去［X］」直接填入目的地並開始導航。  
- **無障礙貫穿**：大按鈕、高對比、震動、電話號逐位朗讀、三語言、TalkBack 相容。  

---

### 第 16 張 — 用戶流程（一頁總覽）
- **環境**：主頁 → 環境識別 → 對準相機 → 聽物體與光線。  
- **出行**：主頁 → 出行協助 → 開始出行 → 說／輸入目的地 → 確認 → 導航頁（路線 TTS ＋ 可步行路徑提示）。  
- **緊急**：主頁 → 長按緊急 3 秒 → 選聯絡人 → 發送短信 ＋ 位置 ＋ 撥號。  
- **語音**：主頁 → 語音助手 → 單次指令或連續對話（如「去 X」）。

### 第 17 張 — 示範預告（將展示什麼）
- **成員 1**：主頁 → 環境識別 → 物體 ＋ 位置 TTS（有時間可加夜間模式）。  
- **成員 2**：主頁 → 出行協助 →「我要去［預設地點］」→ 導航頁 ＋ 可步行路徑 TTS。  
- **成員 3**：主頁 → 文件／貨幣閱讀 或 緊急求助（長按、選聯絡人、短信／位置）。  
- **成員 4**：主頁 → 語音助手（命令 ＋「去 X」）或 手勢快捷。  
- *請按你們實際的「每人 3 分鐘」分工調整。*  

---

### 第 18 張 — 挑戰與應對（可選）
- **挑戰**：物體檢測準確率與延遲。**應對**：多幀融合、最多 2 個物體、置信度調參。  
- **挑戰**：導航偏離。**應對**：距離閾值 ＋ 從當前位置重新規劃。  
- **挑戰**：LLM 不可用。**應對**：回退到關鍵詞匹配執行命令。

## 第二部分：原型示範（每人 3 分鐘）

**規則**：簡報結束後進行示範；每人 3 分鐘；**須在報告前完成系統設定與測試**（設備、網絡、帳號、高德 Key 等）。

### 報告前檢查清單
- [ ] 設備充滿電；螢幕亮度／音量適合場地。  
- [ ] App 已安裝（debug／release）；權限已授予（相機、麥克風、位置，若示範緊急則含短信／電話）。  
- [ ] 語言已設定（若觀眾為英語則設英語）。  
- [ ] 高德 Web Key 有效（用於出行示範）。  
- [ ] 已測試：環境識別、出行（說目的地 → 導航 ＋ 可步行路徑）、OCR 或緊急、語音命令（「打開環境識別」「去［地點］」）。  
- [ ] 已確定每人示範內容，避免重複或切換過久。  

### 建議分工（4 人 × 約 3 分鐘）
- **成員 1**：主頁 → 環境識別 → 展示物體 ＋ 位置 TTS（有餘裕可展示夜間模式）。  
- **成員 2**：主頁 → 出行協助 → 開始出行 → 語音「我要去［預先準備的地點］」→ 導航頁 → 播放一兩句 TTS 指令 ＋ 一句可步行路徑提示。  
- **成員 3**：主頁 → 文件／貨幣（掃文字或紙幣）或 緊急求助（長按、選聯絡人、簡要展示短信／位置／撥號流程）。  
- **成員 4**：主頁 → 語音助手 →「打開環境識別」或「去［地點］」→ 展示正確跳轉；可選一個手勢快捷。  

### 示範時注意
- 口頭用**英語**簡短說明（如 "I'm opening Travel assistance… Now I'll say the destination…"）。  
- 嚴格控制在 **3 分鐘**內；時間緊可省略可選步驟。  
- 若出錯：簡短說明並解釋預期行為（如 "Normally it would speak the next turn"）。

## 第三部分：準備與問答處理

### 材料選擇（評分項）
- **投影片**：標題清晰、文字精簡、一頁一重點；架構圖／用例圖／類別圖在後排也能看清。  
- **圖表**：系統架構、主要使用案例、出行／語音相關簡化類別圖；可選一組用戶流程或時序圖。  
- **證明**：一兩張時程／進度；創新點與挑戰集中呈現。  

### 準備工作
- 用計時器排練（建議 10～11 分鐘，留緩衝）。  
- 若多人報告，分配每人負責的投影片。  
- 若現場示範有風險，準備**簡短備援**（如錄屏或截圖）。  
- 帶好轉接線／線材以配合場地投影。  
- **按規定穿著正式服裝**。  

### 發表要點
- 用**英語**清晰發言；避免照讀投影片。  
- 講架構或流程時指向對應圖表。  
- 進入示範前明確過渡（如 "Now I'll show…"）。  

### 常見問題與建議回答（示例）
- **「為什麼用高德而不是 Google 地圖？」** → 說明地區（如香港／內地）、步行 API 可用性與合規。  
- **「低光環境怎麼處理？」** → 夜間模式：基於置信度判斷、自動降低閾值、TTS 播報。  
- **「用戶偏離路線怎麼辦？」** → NavigationController 檢測距離，從當前位置重新規劃並播報。  
- **「物體檢測準確率如何？」** → 多幀融合下約 85～90%；限制 2 個物體、人／車優先。  
- **「能否離線使用？」** → 環境與導航 TTS 在路線已取得後可離線；路線規劃需網絡；LLM 需網絡，不可用時回退關鍵詞。
