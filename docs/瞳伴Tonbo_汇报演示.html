专案目标

主要目标

打造一站式辅助应用，帮助视障用户更独立地生活，减少对他人依赖。

功能范围

  • 环境感知（实时物体检测与语音播报）
  • 阅读辅助（OCR 文字识别、港币识别）
  • 出行协助（导航 + 可步行路径分析）
  • 紧急求助（一键发位置、短信、拨号）
  • 语音控制（命令 + 连续对话 + LLM）
  • 即时志愿者协助（消息、视讯）

成功标准

语音优先、尽量少触控；支持粤语、英语、普通话；无视力也可用（TTS、震动、大按钮）。

问题背景：视障人群面临的生活问题

  • 出行困难：无法识别路牌、红绿灯、斑马线，易迷路或发生碰撞，独立出行风险高。
  • 环境感知缺失：难以识别周围的人、车、障碍物与光线，影响安全与社交。
  • 信息获取障碍：无法阅读文档、票据、货币面额，日常办事与消费依赖他人。
  • 紧急求助不便：突发情况下一键求助、位置分享、联络家人/999 的能力不足。
  • 操作门槛高：多数 App 依赖视觉介面，视障用户难以独立完成复杂操作。

核心模块（一）：环境感知、阅读、紧急求助

环境感知

摄像头 + SSD MobileNet / YOLO（80 类 COCO）· 多帧融合（连续 3 帧才播报）· 人/车优先、左/中/右位置 · 夜间模式 · TTS 播报

文件与货币

Google ML Kit 中英文 OCR、港币识别 · 分按钮操作 · TTS 朗读 · 独立阅读信件/票据/药品说明；购物时确认找零

紧急求助

长按 3 秒防误触 → 选联络人 → 短信 + 位置 + 拨号（含 999）· 电话号逐位数字朗读（粤/英/普）

出行协助 — 定位与价值

作用：从「说目的地」到「跟着语音走」的全流程，实现「听得懂的导航」。

  • 路线规划：高德步行 API，获取分段步骤与总距离/时长
  • 分段 TTS 导航：播报「前行 XX 米」「左转」「右转」；接近转弯约 20 米内自动播报下一段
  • 偏离重规划：实时检测偏航，从当前位置重新规划并播报
  • 可步行路径分析：相机 + YOLO 检测前方障碍，分析左/中/右受阻占比，播报「前方畅通」「中间受阻请向左/右慢行」「前方有斑马线」「前方红绿灯」等

意义：视障用户仅凭语音即可完成从输入目的地到步行导航的全过程，减少碰撞与迷路。

出行协助 — 技术实现

路线规划

RoutePlanner 调用高德 Web 服务（地理编码 + 步行路线）· NavigationController 周期定位、偏离检测、触发重规划

导航页

NavigationActivity + 高德 MapView（蓝点、已走/未走路线）· TTS 分段播报「前行 XX 米」「左转」等，约 20 米内自动播下一段

语音入口

VoiceCommandManager 解析「我要去 XXX」→ TravelParseResult → StartTravelActivity 填入目的地

可步行路径

TravelDetectionController + CameraX + YOLO 检测行人/车/障碍/斑马线/红绿灯 · WalkAssistManager 左/中/右分析 → 路况短语播报 · OverlayView 绘制检测框

创新点与亮点总结

  • 「听得懂的出行」全链路：语音目的地 → 高德步行路线 → 分段 TTS + 转弯提醒 + 偏离重规划 + 可步行路径分析（障碍、斑马线、红绿灯），实现「不看地图也能安全走」的闭环。
  • 多帧融合 + 优先级播报：环境检测连续 3 帧稳定才播报，人/车优先、左中右位置描述，减少误报与信息过载。
  • 夜间模式自动适配：根据检测置信度推断低光，自动降低阈值并播报「夜間模式已開啟」，无需用户操作。
  • 语音与出行深度结合：「我要去天水围」等自然句直接进入路线与导航，降低操作步骤。
  • 无障碍贯穿全产品：大按钮、高对比、震动手感、电话号逐位朗读、三语言统一、TalkBack 兼容。
